# 画面設計書 — 売掛金 可視化・債権回収管理アプリ

> 前提: 利用者5名（非技術・経理中心） / **デスクトップ運用**（オフィス）/ 回収業務中心
> データモデルは `docs/db_design.md` を参照（Client / Invoice / Receipt / Allocation / CollectionAction / ImportBatch / MonthlyClose）。

---

## 1. 設計方針

- **「今日やること」が一目で分かる**：ログイン直後に要フォロー先・延滞が見える。
- **回収業務の動線を最短に**：顧問先詳細＝元帳・未納・督促記録が1画面に集約。
- **手作業の置換**：スプレッドシート転記 → 月次レポート出力で消す。
- **非技術でも迷わない**：左ナビ固定、専門用語を避けた表示、操作は「見る・記録する・出力する」が中心。

## 2. 画面一覧

| # | 画面 | 目的 | 主データ |
|---|------|------|----------|
| S1 | ダッシュボード | 全体把握＋今日やること | 集計／CollectionAction／ImportBatch |
| S2 | 顧問先一覧（回収ワークリスト） | 回収対象の俯瞰・絞り込み | Client＋残高 |
| S3 | **顧問先詳細**（中核） | 元帳・未納請求・督促を集約 | Invoice/Receipt/Allocation/CollectionAction |
| S4 | 督促ワークリスト | 要フォロー（次回予定到来）の一覧 | CollectionAction |
| S5 | 消込（入金引当） | FIFO自動の確認＋例外の手動付替 | Allocation |
| S6 | 月次残高確定・レポート | 月末締め・報告出力 | MonthlyClose |
| S7 | 取込状況 | 自動同期の監視 | ImportBatch |
| S8 | マスタ／ユーザー管理 | 顧問先・担当者の管理 | Client/User（Django admin流用） |

## 5. 各画面詳細

### S1. ダッシュボード

```
┌─ KPI ───────────────────────────────────────────────────────────┐
│ 売掛金総残高     未入金請求    期日超過残高      要フォロー(本日) │
│   ¥4,820,000      23 件        ¥1,150,000          5 件           │
└──────────────────────────────────────────────────────────────────┘
┌─ エイジング ──────────────────────┐ ┌─ 要フォロー先（本日以前）──────┐
│ 0-30日   ¥2,300,000  ████████      │ │ ○○商事  ¥330,000  90日超 担当:佐藤│
│ 31-60日  ¥1,370,000  █████         │ │ △△工業  ¥210,000  61日   担当:鈴木│
│ 61-90日  ¥  600,000  ██            │ │ □□物産  ¥120,000  45日   担当:佐藤│
│ 90日超   ¥  550,000  ██            │ │ … → 全件はワークリストへ        │
└────────────────────────────────────┘ └──────────────────────────────────┘
┌─ 滞留の大きい先 TOP5 ─────────────┐ ┌─ 直近の督促アクション ──────────┐
│ ○○商事 90日超 ¥550,000            │ │ 06/01 鈴木 △△工業 電話 入金約束 │
│ …                                  │ │ 05/31 佐藤 ○○商事 メール 督促状  │
└────────────────────────────────────┘ └──────────────────────────────────┘
```
- **要素**: KPIカード4つ／エイジング棒グラフ／要フォロー先（`next_action_date ≤ 本日`）／滞留TOP5／活動ログ。
- **クリック**: 各行 → S3 顧問先詳細。
- **データ**: 残高=Σ請求−Σ入金、エイジング=Allocation残額、要フォロー=CollectionAction、活動=最新CollectionAction。

### S2. 顧問先一覧（回収ワークリスト）

```
[絞込] 担当者[全▼] エイジング[全▼] ☑残高あり ☐要フォローのみ  🔍[          ]   [CSV出力]
┌──────────┬──────┬───────────┬─────────┬──────────┬──────────┬────────┐
│ 顧問先   │担当者│ 売掛残高  │最長延滞 │90日超残高│最終督促  │次回予定│
├──────────┼──────┼───────────┼─────────┼──────────┼──────────┼────────┤
│ ○○商事  │佐藤  │  ¥550,000 │ 92日 🔴 │ ¥550,000 │05/31 メール│06/05  │
│ △△工業  │鈴木  │  ¥210,000 │ 61日 🟡 │       ¥0 │06/01 電話 │06/08  │
│ □□物産  │佐藤  │  ¥120,000 │ 45日    │       ¥0 │  ―       │  ―    │
└──────────┴──────┴───────────┴─────────┴──────────┴──────────┴────────┘
                                          ソート: 残高▼ / 延滞日数 / 次回予定
```
- **絞込**: 担当者・エイジング区分・残高あり・要フォローのみ・フリーワード。
- **延滞バッジ**: 60日超🟡 / 90日超🔴。
- **行クリック** → S3。**CSV出力**で一覧をエクスポート。

### S3. 顧問先詳細（★中核画面）

```
┌─ ○○商事 (C001)  担当:佐藤 ────────────────────────────────────────┐
│ 現在残高 ¥550,000   最長延滞 92日 🔴   ステータス: 督促中            │
│                          [督促を記録] [消込を調整] [この先のレポート]│
└──────────────────────────────────────────────────────────────────────┘
┌─ ① 未入金請求 ───────────────────────────────────────────────────────┐
│ 請求日   期日     請求額     残額      経過    区分                    │
│ 03-31    04-30    ¥330,000  ¥330,000  92日🔴  61-90超                  │
│ 04-30    05-31    ¥220,000  ¥220,000  31日    31-60                    │
│                                  ☐未納のみ表示                        │
└──────────────────────────────────────────────────────────────────────┘
┌─ ② 取引履歴（得意先元帳） ──────────────────────────────────────────┐
│ 期間 [2026-04-01]〜[2026-06-01]                                       │
│ 日付   区分  摘要           増加(+)    減少(−)   残高     登録者       │
│ 繰越                                            ¥0                     │
│ 03-31  請求  3月分顧問料    ¥330,000            ¥330,000 system(board) │
│ 04-30  請求  4月分顧問料    ¥220,000            ¥550,000 system(board) │
│ 05-20  入金  普通預金                  ¥0       ¥550,000 (引当なし)     │
└──────────────────────────────────────────────────────────────────────┘
┌─ ③ 督促履歴 ─────────────────────────────────────────────────────────┐
│ 06/01 鈴木 電話    不在・折返し依頼          次回:06/05               │
│ 05/31 佐藤 メール  督促状送付                次回:06/05  [編集]       │
└──────────────────────────────────────────────────────────────────────┘
```
- **①未入金請求**（§5.2）: 請求ごとの残額・経過日数・エイジング。「いつの債権が残っているか」。
- **②得意先元帳**（§5.1）: 繰越残高＋請求/入金の残高累計。期間フィルタ。
- **③督促履歴**（CollectionAction）: 時系列。`[編集]` は履歴(simple-history)に残る。
- **［督促を記録］モーダル**:
  ```
  日時[2026-06-01 14:30]  手段[電話▼]  ステータス[督促中▼]
  内容[                                   ]
  結果[                                   ]
  次回フォロー日[2026-06-05]              [キャンセル] [記録する]
  ```
- **データ**: actor=ログインユーザー（自動）、created_by 記録。

### S4. 督促ワークリスト（要フォロー）

```
[絞込] ☑自分の担当(佐藤) ☐期日超過のみ        並び: 次回予定昇順
┌──────────┬───────────┬─────────┬──────────────────┬────────┐
│ 顧問先   │ 売掛残高  │最長延滞 │ 前回督促          │次回予定│
├──────────┼───────────┼─────────┼──────────────────┼────────┤
│ ○○商事  │  ¥550,000 │ 92日🔴  │05/31 メール 督促状│06/05🔴 │ ←期限超過
│ □□物産  │  ¥120,000 │ 45日    │05/28 電話 入金約束│06/03   │
└──────────┴───────────┴─────────┴──────────────────┴────────┘
```
- `next_action_date ≤ 本日` を上位・赤表示。担当者で自分の分に絞れる。
- 行クリック → S3（督促記録へ）。

### S5. 消込（入金引当）

```
顧問先[○○商事 ▼]                       [FIFO自動引当を再実行]（手動分は保護）
┌─ 未引当の入金 ──────────────┐   ┌─ 未納の請求 ──────────────────────┐
│ 日付   金額     未引当       │   │ 請求日  期日    請求額   残額       │
│ 05-20 ¥330,000 ¥330,000 ◀──┼─┐ │ 03-31  04-30  ¥330,000 ¥330,000 ◀─┐│
│ 05-28 ¥220,000 ¥  0    自動 │ └─┤→引当                              ││
│                             │   │ 04-30  05-31  ¥220,000 ¥  0  自動 ◀┘│
└─────────────────────────────┘   └────────────────────────────────────┘
 引当方式: 自動(FIFO) / 手動  選択して [引当を変更] / [引当を解除]
 ※未引当入金 ¥0（前受なし）   過引当チェック OK
```
- 左=入金、右=請求。FIFO自動引当の結果を表示し、係争・指定入金は手動で付替（method=manual、自動再実行で保護）。
- 過引当を禁止（Σ引当 ≤ 各金額）。手動操作は履歴に残る。

### S6. 月次残高確定・レポート

```
対象月 [2026-05 ▼]   状態: 未確定          [残高を確定する] [レポート出力▼]
┌──────────┬──────────┬───────────┬───────────┬──────────┬──────────┐
│ 顧問先   │ 期首残高 │ 当月請求  │ 当月入金  │ 期末残高 │ 90日超   │
├──────────┼──────────┼───────────┼───────────┼──────────┼──────────┤
│ ○○商事  │ ¥220,000 │  ¥330,000 │       ¥0  │ ¥550,000 │ ¥330,000 │
│ △△工業  │ ¥210,000 │  ¥210,000 │ ¥210,000  │ ¥210,000 │       ¥0 │
├──────────┼──────────┼───────────┼───────────┼──────────┼──────────┤
│ 合計     │ ¥…       │ ¥…        │ ¥…        │¥4,820,000│¥550,000  │
└──────────┴──────────┴───────────┴───────────┴──────────┴──────────┘
 MF会計の売掛金残高と照合: [MF残高を入力] → 差異チェック
```
- 「月末+5日に残高確定→報告」を置換。`[残高を確定する]`で MonthlyClose スナップショット作成（closed_by 記録、以後ロック）。
- **レポート出力**: 顧問先別残高・エイジング・督促状況を CSV/Excel/PDF。
- **MF照合**: MFの売掛金残高を入力し合計差異を確認（月次照合の証跡）。

### S7. 取込状況（ImportBatch）

```
[手動で再取込▼] board / MF入金 / NSS
┌──────────────┬────────┬───────┬───────┬───────┬─────────┐
│ 実行日時     │ ソース │ 取込  │スキップ│エラー │ 状態    │
├──────────────┼────────┼───────┼───────┼───────┼─────────┤
│06/01 06:00   │ board  │  12   │   0   │  0    │ ✅成功  │
│06/01 06:01   │ MF入金 │   8   │   1   │  2    │ ⚠一部   │ [詳細]
│05/31 06:00   │ NSS    │   5   │   0   │  0    │ ✅成功  │
└──────────────┴────────┴───────┴───────┴───────┴─────────┘
  [詳細]→ エラー行のログ（例: 3行目 顧問先コード"X9"未登録 → スキップ）
```
- 自動同期の監視。エラー/スキップ理由を表示し、修正後に手動再取込。

### S8. マスタ／ユーザー管理

- **顧問先マスタ**（コード・名称・担当者・締め日・サイト・有効/無効）と**ユーザー（担当者）**は、当面 **Django管理画面**を流用。非技術でも一覧・編集は可能な範囲。利用頻度が上がれば専用画面化。

## 6. 権限・ロール（5名想定）

| ロール | できること |

| 一般（担当者） | 全画面の閲覧、督促記録、消込の手動調整 |

## 7. 実装メモ（Django対応）

| 画面 | View / Template | 備考 |
|------|-----------------|------|
| S1 | `dashboard` | 集計クエリ。重ければ MonthlyClose や集計キャッシュを併用 |
| S2 | `client_list` | フィルタ・ソート・CSV出力 |
| S3 | `client_detail` | 旧 `sub_ledger` の残高累計ロジックを流用（§5.1） |
| 督促モーダル | `collection_action_create/update` | actor=request.user |
| S4 | `followup_worklist` | `next_action_date` 抽出 |
| S5 | `allocation` | FIFO自動＋手動付替。`reconcile_fifo()` バッチ |
| S6 | `monthly_close` | スナップショット＋エクスポート |
| S7 | `import_status` | ImportBatch 一覧＋手動再取込トリガ |
| S8 | Django admin | 顧問先・ユーザー |

- 既存テンプレート（`base.html` 等）と Bootstrap 的なシンプルUIを踏襲。
- 数値は3桁区切り・右寄せ、延滞は色（🟡60日超/🔴90日超）で視認性を確保。
